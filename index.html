Frog vs. Fireball

by: Darren Shan

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I decided to make this game because I wanted to do something simple and familiar, yet still challenging enough that it wouldn't be too easy. Originally, I had planned on making a much more complicated game, but I had some early struggles with that project and decided to start fresh. My idea for this game came when I thought back to some old Scratch projects I had seen/done before, and I was inspired to create something similar, using Java. Additionally, this project had several similar aspects as League Invaders, helping make the process slightly easier.

2. What I Learned

The majority of the feedback I got was positive feedback about what the person liked about the game. One suggestion that I got was to make it so that when the character reached the edge of the window, to stop them from moving past the edge or teleport to the other side. I went back and changed it so that the character's move method would work unless they were at certain x-values which would represent the edge of the window.

3. Architecture of My Game

My game has 7 different classes and they are mainly connected to the Game_Runner, Game_Panel, Game_Object, and Object_Manager classes. These classes are responsible for developing and implementing the game window (display), settings, and variables/methods from other classes. The classes titled Character, Fireball, and Token deal with the design, function, and implementation of the given element.

4. Technical Implementation

The part of my game I am most proud about is the code related to the Fireball and Token classes. My idea was that tokens would spawn less frequently than fireballs, and hitting a token would increase your score, while hitting a fireball would kill/harm you. To do this, I set two different timers and spawn methods which would cause each different element to spawn at different rates. Furthermore, I had to set up the collision methods to identify which object made contact with the character and what to do (-1 life or +1 point) as a result. Then, I had the idea of giving the character 3 lives and making them "invincible" while dashing. For this part, I had to add new conditions to the collision statements to determine whether the character was actively dashing and how many lives they have left.

5. Most Challenging Part

The most challenging part of this project was figuring out how to actually turn my ideas into working code and deal with errors/flaws in the code. For the most part, I had clear, precise ideas on how the mechanics for my game would work, but only some of them aligned with the code I had to write in order to make my game. In the situations where I didn't know how to code something or the code which I had written didn't work, I would ask for help or try and logically understand the issue. Eventually, through time and/or lots of trial and error, I would fix each problem until my game was complete and ready.

6. Future Fixes and Updates

I would add some more features like power-ups, new/upgraded obstacles, and possibly some other modes. I would also maybe make the code more "efficient" and improve/alter the ways in which certain tasks are carried out. For example, I would try to find a new system for the dashing mechanic or making it so that fireballs and tokens can't overlap when they spawn.
